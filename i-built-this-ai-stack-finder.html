Behind the build

I built a free AI tool recommender for e-commerce sellers — here's how and why

Most "best AI tools" articles give you the same 12 tools in a list and tell you to figure it out yourself. I wanted something better — a tool that asks you five questions and tells you specifically what to use for your situation. So I built one. Here's the whole story.

The problem I kept running into

When I started writing reviews of AI tools for e-commerce sellers, I noticed the same pattern every time. Someone would read a review of Jasper, think "this sounds good," sign up, realise it wasn't solving their actual problem, and cancel within a month.

The issue wasn't that Jasper is bad. The issue is that a solo Etsy seller who needs help writing customer message templates has completely different needs from a Shopify store owner with 200 SKUs who needs to write product descriptions at scale. Recommending the same tool to both of them isn't helpful — it's lazy.

I wanted to build something that asked a few simple questions — what platform, what stage, what's your biggest headache, what's your budget — and then gave a specific, honest answer. Not a list of every tool that might possibly be relevant. An actual recommendation.

What I built

The AI Stack Finder is a five-question quiz that outputs a personalised AI tool stack. It's free to use, takes about two minutes, and doesn't ask for your email.

The five questions are:

  • What platform do you sell on? (Shopify, Etsy, Amazon, multiple, or not yet)
  • Where is your store right now? (revenue stage)
  • What's your biggest headache? (writing, support, email, SEO, photos)
  • What's your monthly tool budget? (from free to $150/month)
  • How comfortable are you with technology?

Based on those answers it recommends three to five specific tools, ranked by priority — which one to start with, which to add next, and which to consider when you're scaling. Each recommendation explains exactly why it applies to that person's specific situation, not a generic description.

How it works technically

The whole thing is built in plain HTML, CSS, and JavaScript — no framework, no build step, no dependencies. The quiz UI is a single HTML file that lives in the same repo as this site and deploys automatically to GitHub Pages when I push to main.

For the AI-powered recommendations, I built a Cloudflare Worker — a small serverless function that runs at Cloudflare's edge for free. The worker receives the quiz answers, formats them into a prompt, sends them to the Anthropic API (Claude), and returns a structured JSON recommendation. The API key lives securely in the worker's environment variables and never touches the frontend.

The prompt instructs Claude to recommend only from a curated list of tools I've actually reviewed, to never recommend paid tools to someone who selected a zero budget, and to always prioritise the tool that addresses the user's stated pain point first. It returns structured JSON that the frontend renders into a clean results card.

The clever part: if the API call fails for any reason — network error, rate limit, anything — the tool falls back silently to a rule-based recommendation engine built in JavaScript. Users get a good result either way. The fallback handles edge cases like Etsy sellers (who get eRank recommended regardless of what their pain point is, because Etsy SEO is different from everything else) and zero-budget users (who only see tools with genuinely useful free plans).

The tech stack at a glance

  • Frontend: Plain HTML, CSS, JavaScript — no framework
  • Hosting: GitHub Pages (free)
  • Serverless function: Cloudflare Workers (free tier)
  • AI: Anthropic API — Claude Haiku for fast, cheap inference
  • Database: None — stateless, no user data stored
  • Total monthly cost: $0

The Anthropic free credits cover several hundred quiz completions before you'd need to pay anything. At Claude Haiku's pricing, even after free credits run out, each quiz completion costs roughly $0.001. You'd need 10,000 completions a month before it cost more than $10.

What I learned building it

The hardest part wasn't the code. It was writing the system prompt. Getting Claude to return clean, structured JSON consistently — without wrapping it in markdown code blocks, without hallucinating tool names, without recommending paid tools to free-budget users — took more iteration than anything else.

The solution was being extremely specific in the prompt. Rather than saying "recommend appropriate tools," I gave it an explicit list of every tool it was allowed to recommend, the exact JSON schema it had to follow, and a list of hard rules (no paid tools for zero-budget users, always include eRank for Etsy sellers, etc.). Constrained prompts produce consistent outputs.

The other lesson: build the fallback first. I built the rule-based JavaScript recommendation engine before I connected the API. This meant the tool was useful and live before I'd written a single line of API code. By the time I added the AI layer, the fallback was already handling edge cases well, which made the AI output easier to validate — I could compare the two and see where Claude was doing better or worse than the rules.

A note on affiliate links

The tool recommends Tidio and Omnisend with affiliate links — those are the two programs I'm currently enrolled in. Every other tool in the recommendations links to the tool's own website with no tracking. I didn't build the quiz to funnel everyone toward the affiliate tools — the recommendations are based on what actually fits the user's answers, and both Tidio and Omnisend genuinely are the right tools for a lot of situations.

What's next for the tool

Right now the tool is stateless — it doesn't remember anything between sessions, doesn't store results, and doesn't require an account. That's intentional. The fastest path to something useful was to skip the database entirely and build something that works for anonymous users immediately.

As the site grows, I'd like to add a "save my stack" feature that emails your results to you — which also builds the email list. That requires a database (Supabase free tier) and an email provider, both of which I've kept out of scope for the first version deliberately.

I also want to expand the tool list as I review more products. Right now it covers the core categories well but is thin on analytics and automation tools. Those will be added as I write the corresponding reviews.

If you want to build something similar

The full approach — static site, Cloudflare Worker, Anthropic API — is genuinely free to run at low volume and surprisingly capable. If you want to build a similar tool for your own niche, the pattern works for any kind of recommendation engine: input a few parameters, format them into a prompt, return structured JSON, render it.

The Cloudflare Worker approach is important for keeping your API key secure. Never put an Anthropic API key in frontend JavaScript — anyone can view source and steal it. The worker acts as a proxy that holds the key server-side and only passes the formatted answers to the API.

The whole quiz is about 400 lines of HTML, CSS, and JavaScript. It doesn't need React, it doesn't need a build pipeline, and it doesn't need a database. Sometimes the simplest approach is the right one.
